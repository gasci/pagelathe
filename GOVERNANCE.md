# Governance

pagelathe uses a lightweight maintainer model.

- **Maintainers** (see [MAINTAINERS.md](./MAINTAINERS.md)) review and merge PRs, triage
  issues, and cut releases. Changes merge after at least one maintainer approval and green CI.
- **Decisions** are made by lazy consensus on issues/PRs. If consensus is unclear, the
  maintainers decide; the project lead (@gasci) breaks ties.
- **Becoming a maintainer:** sustained, high-quality contributions (code, reviews, section
  authoring) earn an invitation from the existing maintainers.
- **Roadmap** is tracked in [ROADMAP.md](./ROADMAP.md) and the issue tracker.
